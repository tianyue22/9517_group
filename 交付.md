## 1. 传统学习的方法

C1:加入ExG/LBP能提升F1与loU

C2:后处理主要提升 precision（去噪）但可能损失 recall

C3:在 noise/blur/dark 下，改进版比基础版更稳

**Part 1 特征消融**

1. F0: RGB only
2. F1: RGB + ExG
3. F2: RGB + LBP
4. F3: RGB + ExG + LBP（当前）
5. 结论看 F1/IoU 主指标，并报显著性。

结果：一张表，行：F0, F1, F2, F3，列：Precision, Recall, F1, IoU（都写 mean±std）

**Part 2 后处理消融（回答“precision-recall权衡来自哪”）**

1. P0: 无后处理
2. P1: 仅 morphology(open+close)
3. P2: 仅小连通域过滤
4. P3: morphology + 小连通域过滤（当前）
5. 再扫 MIN_COMPONENT_AREA（如 30/80）
6. 重点展示 Precision↑ / Recall↓ 是否稳定。

结果：一张表，行：P0, P1, P2, P3, P3_area30, P3_area80，列：Precision, Recall, F1, IoU（mean±std）。第六点可以画散点图（可放可不放，看论文能不能塞得下）

**Part 3 鲁棒性实验（回答“改进版是否更稳”）**

1. 扰动类型：noise/blur/dark（你已有）
2. 强度分级：轻/中/重（例如噪声 sigma 10/20/30；blur k=3/5/7；dark 0.8/0.6/0.4）
3. 比较对象：baseline vs best_ablation_variant
4. 输出：退化曲线 F1(or IoU) vs severity。

一张图，x：severity（clean, light, medium, severe）y：F1。两条线：baseline、best_ablation_variant。扰动维度做3 个小子图：noise、blur、dark

**Part 4 失败图三张**

1. 人工标注失败类型：弱光、土壤高亮、细叶断裂、杂草混淆、遮挡
2. 给出“对应改进建议”（如阴影补偿、边界细化），也可以放在最后的discuss部分

**Part 5 统计与结论（把结果变成证据）**

- “C1 成立/不成立：指标变化 + p值”
- “C2 成立/不成立：Precision-Recall 权衡是否显著”
- “C3 成立/不成立：鲁棒性退化斜率是否更缓”



### 2. 机器学习方法

C4：颜色+纹理+边缘的组合特征优于单一特征
C5：去噪在噪声/模糊场景可提升稳定性，但可能带来边界损失
C6：后处理可提升 mask 连贯性与 precision

Part 1 特征工程消融

1. M0：颜色特征（RGB/HSV + ExG）
2. M1：颜色 + 纹理（LBP）
3. M2：颜色 + 边缘（Sobel/Canny梯度统计）
4. M3：颜色 + 纹理 + 边缘（当前完整版）
5. 模型固定（如 RF），其余流程固定。
   **输出：一张表（M0-M3，列 P/R/F1/IoU/Time，mean±std）**

Part 2 去噪策略对比

1. D0：无去噪
2. D1：Gaussian
3. D2：Median/Bilateral（二选一）
4. 只在 M3 特征上比较。
   **输出：一张表（D0-D2，列 P/R/F1/IoU）**

Part 3 模型对比（体现“机器学习”）

1. RF（基线）
2. SVM（RBF）或 XGBoost（二选一）
3. 使用同一特征（建议 M3）和同一后处理。
   **输出：一张表（Model，列 P/R/F1/IoU/Train time/Test time）**

Part 4 后处理消融

1. H0：无后处理
2. H1：morphology
3. H2：连通域过滤
4. H3：morphology + 连通域过滤（当前）
   **输出：一张表（H0-H3，列 P/R/F1/IoU）**

Part 5 鲁棒性图（与你第一部分风格一致）

1. 扰动：noise/blur/dark，轻中重
2. 比较：ML-baseline vs ML-best
   **输出：一张图（3子图，x=severity，y=F1，2条线）**
